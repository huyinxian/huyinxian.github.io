Unity3D学习笔记（2）——C#脚本

发表于 2018-04-14   |   分类于 Unity3D   |  

本章介绍一些脚本相关的注意事项

本文仅供个人记录和复习，不用于其他用途

脚本基础

C# 脚本要取有意义的名字，类名要与脚本名一致。在脚本中默认会有 Start() 和 Update() 两个方法，分别代表了脚本启动时执行的方法以及每帧中会执行的方法。这种发生某种事件时调用的方法还有很多，之后会介绍。

脚本是依附于游戏对象的，事件和操作的对象也是当前脚本所在的游戏对象。脚本只能依附于游戏对象或者由其他脚本调用才会运行，一个脚本可以放到多个游戏对象上，是多个实例。一个脚本的多个实例和其他脚本之间互不干扰。

命名空间

C# 脚本不支持命名空间，脚本在 Project 中的显示名称不要加 .cs。所有的类都继承自 MonoBehaviour，跟具体游戏对象无关的脚本可以放到摄像机 MainCamera 中。

脚本初始化

如果想要设置脚本的逻辑路径，可以使用[AddComponentMenu(“a/b”)]，即表示在 a 文件夹中有 b 脚本。要注意的是，对于相同的类，不管你如何命名，都仅仅只是改变逻辑上的路径，不会影响类的调用。

类的初始化不要放到构造函数中，要放到 Start() 方法中。实例化的步骤则不需要我们关心，在脚本添加到游戏对象上时，U3D 就已经帮我们完成了实例化的步骤。另外，get/set 方法将不被识别，想要修改属性可以直接在 Inspector 中修改，带有 public 修饰符的字段即为属性，否则将自动视为私有（private）。
